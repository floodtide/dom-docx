## Style source: inline vs computed-oracle vs computed-native (dom-docx)

Run: `tsx tools/style-source-benchmark.ts` (requires fresh `npm run test:suite` baseline in `output/suite/results.json`).

Three style-resolution lanes: **inline** (default `style=""` resolver, pure JS), **computed-oracle** (`convertHtmlToDocx(html, { styleSource: "computed", browser })`, spawns its own Playwright page per call), **computed-native** (`convertHtmlToDocx(html, { styleSource: "computed", page })`, reads styles from an already-rendered page — the in-browser deployment lane).

| Metric | inline | computed-oracle | computed-native | native − inline |
|--------|-------:|----------------:|----------------:|----------------:|
| XML schema pass | 28 / 28 | 28 / 28 | 28 / 28 | 0 |
| Avg **adjusted** visual | 88.66% | 88.32% | **88.32%** | **−0.35 pp** |
| Avg editability | 99.64 | 99.64 | 99.64 | 0.00 |
| Avg engine score | 92.99 | 79.03 | **91.80** | −1.20 |
| Avg compile | 21.5 ms | ~632.2 ms | **~26.7 ms** | +5.2 ms |

Guard results:

- **Parity guard** (`tsx tools/computed-parity-guard.ts`): oracle and native emit byte-identical normalized `word/*.xml` for identical HTML — **30/30 byte-identical**.
- **Browser bundle guard** (`npm run build:browser && tsx scripts/browser-spike.ts && tsx tools/browser-build-parity.ts`): esbuild IIFE (`dom-docx/browser`) parity with the Node computed-native path — **30/30 byte-identical**.
- **Inline path guard** (`npm run test:inline-guard`): default vs explicit `styleSource: "inline"` — **28/28 equivalent**.

> Playwright is only the *test host* for the native lane (a headless browser exercising the in-page code path under CI); the real browser-side deployment calls `getComputedStyle` on the live `document` with no Playwright and no server. The oracle lane is the genuine server-side path where Playwright is a runtime dependency.

## CSS cascade suite (stylesheet / class selectors)

Run: `tsx tools/css-cascade-runner.ts` · Cases: `tools/css-cascade-cases.ts` · Output: `output/css-cascade/results.json`

Separate from the main loop — HTML uses `<style>` blocks and `class`/`#id` selectors with no matching inline `style=""` on styled nodes (except `stylesheet-inline-wins`).

| Metric | inline | computed | Δ |
|--------|-------:|---------:|--:|
| Avg adjusted visual | 68.04% | 82.24% | **+14.20 pp** |
| Cases passed | — | 7 / 8 | — |

| Case | Δ (computed − inline) |
|------|----------------------:|
| `stylesheet-section-theme` | +49.2 pp |
| `stylesheet-descendant-table` | +48.8 pp |
| `stylesheet-strong-em` | +7.0 pp |
| `stylesheet-p-color` | +4.0 pp |
| `stylesheet-inline-wins` (control — inline overrides class) | ~0 pp |

Per the parity guard above, computed-native produces byte-identical output to computed-oracle, so this cascade win applies to the browser-side lane at native speed (~27 ms/case) rather than the oracle's ~630 ms.
